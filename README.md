# aix-agents

`aix-agents` is a portable, file-based agent skill pack system for projects that support `AGENTS.md`.

When installed into another project, this pack should live at:

```txt
.agents/aix-agents/
```

It provides:

- reusable agent skills in `skills/`
- reusable rule templates in `rules/`
- runtime active rules in `.aix/rules/` (created by `00-aix-init.md`)
- runtime artifacts under `.aix/artifacts/`
- default task/progress/context tracking through `planning-with-files`

## Design Principle

Agents should not depend on hidden memory or direct skill-to-skill communication.

Instead, they coordinate through files.

The only default artifact boundary is:

```txt
.aix/artifacts/
```

Each skill may create its own internal structure under the active task artifact folder.

## Project Layout

```txt
skills/           reusable skills (source)
rules/            reusable rules (source)
.aix/rules/       active project rules (runtime, generated by init)
.aix/artifacts/   runtime artifacts
AGENTS.md         agent entrypoint/router
```

## Installation

Install by cloning this repository into the target project's `.agents/aix-agents` folder:

```bash
git clone https://github.com/mdhb2/aix-agents.git .agents/aix-agents
```

If `.agents/` does not exist yet, create it first:

```bash
mkdir -p .agents
git clone https://github.com/mdhb2/aix-agents.git .agents/aix-agents
```

After clone, use source pack from `.agents/aix-agents/`:

- skills from `skills/`
- rules from `rules/`

Then run `rules/00-aix-init.md` in target project to create `.aix/rules/` and `.aix/artifacts/`.

After init, use `AGENTS.md` to include `.aix/rules/RULES.md` as active rule router.

## Post-init Verification

After `00-aix-init.md` runs, verify runtime files:

```txt
.aix/
.aix/artifacts/
.aix/rules/
.aix/rules/RULES.md
.aix/rules/README.md
.aix/rules/04-write-boundaries.md
.aix/rules/05-communication-caveman.md
.aix/.initialized-by-aix-agents
```

Verify marker file is valid JSON and includes:

- `version`
- `initializedAt`
- `source`
- `copyMode` (expected: `copy_if_missing`)
- `rulesRegistry`

## Default Task Tracking

The `planning-with-files` skill is the default mechanism for:

- task state
- progress updates
- important decisions
- context preservation
- resumable work

## External Skill Source

The default planning skill should be based on:

```txt
https://github.com/OthmanAdi/planning-with-files/tree/master/skills/planning-with-files
```

If the skill source is not vendored yet, leave `skills/planning-with-files/README.md` with integration notes.
